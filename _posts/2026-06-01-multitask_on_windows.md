不管是传统的软件开发中等待编译的时间，还是在更现代的软件开发过程中等待ai执行任务的时间，实际上一直都会有需要等待任务执行的情况。这实际上比较类似计算机在面对更慢的外部硬件时的情况，比如类似时间片轮转的技术。

对应的，在这样的场景当中，需要的就是可以高效的进行上下文切换，也就是高效的将一个任务对应的窗口布局切换到另一个任务对应的窗口布局。

## 虚拟桌面

Windows 本身提供了虚拟桌面的功能，具体来说可以通过`win+tab`进入任务视图，在其中可以创建桌面，并将窗口分配到指定的桌面。

|     功能     |        快捷键         |
| :----------: | :-------------------: |
| 打开任务视图 |       `win+tab`       |
|   切换桌面   | `win+ctrl+left/right` |

## PowerToys: Workspaces

虚拟桌面虽然可以让已经存在的窗口比较好的被管理，但是对于重启或是因为电脑性能受限而需要关闭窗口的情况，用户仍然需要手动的去启动一个一个窗口。PowerToys提供了一个插件Workspaces，它的主要职责就是录制当前屏幕布局，并且允许配置软件的启动参数来直接地打开对应的项目。

|        功能         |    快捷键     |
| :-----------------: | :-----------: |
| 打开Workspace编辑器 | `win+ctrl+\`` |

## Windows 窗口快捷键

对于窗口较多的情况，单纯的将窗口用某种布局放置在屏幕上实际上对视图阅读某一个窗口当中的内容并不友好，实际上windows提供了一些快捷键来快速的全屏和还原某一个窗口。

|    功能    |                快捷键                |
| :--------: | :----------------------------------: |
|  选择窗口  | `alt+tab`之后再按tab即可继续切换窗口 |
| 最大化窗口 |               `win+up`               |
|  还原窗口  |              `win+down`              |
